# 毫米波雷达—FAST-LIO 速度紧耦合:全流程原理与详细推导

> 适用工程:`radar_ego_velocity`(自速度估计) + `FAST_LIO_GLOBAL`(LIO 紧耦合)
> 目标场景:地下管廊 / 长直隧道 / 长走廊等**几何退化**环境下,抑制激光里程计沿轴向的定位漂移。
> 对应源码:
> - 第一级 `src/radar_ego_velocity_node.cpp`(多普勒 → 自速度)
> - 第二级 `FAST_LIO_GLOBAL/src/laserMapping.cpp` 的 `fuse_radar_velocity()`(自速度 → ESKF 观测更新)

---

## 0. 为什么需要雷达

FAST-LIO 的状态来自 IMU 前向传播 + 激光扫描匹配(point-to-plane)的迭代卡尔曼更新。
在长直隧道/管廊中,激光点云**沿轴向没有可约束的几何特征**(墙面平行、断面自相似),
point-to-plane 的信息矩阵在轴向上**秩亏**,导致:

- 该方向的扫描匹配残差对位姿不敏感 ⇒ 观测约束缺失;
- ESKF 后验协方差 `P` 在沿轴速度/位置分量上**塌缩**(过度自信);
- IMU 的加速度零偏与重力投影误差在轴向上无人纠正 ⇒ **里程计沿轴匀速漂移**。

毫米波雷达提供的是**多普勒径向速度**,它**不依赖几何特征**:只要环境里有静止反射点
(墙壁、地面、管壁),就能直接测出载体的运动速度,恰好补上激光缺失的轴向约束。

```
SR71 雷达 (串口/CAN)
  └─ sr71_serial_node ──/radar_scan──► radar_ego_velocity_node ──/radar_velocity──► FAST-LIO
     (解析硬件帧)        (RadarScan)    (RANSAC + 最小二乘估自速度)  (TwistStamped)   (ESKF 速度观测更新)
```

整个系统分两级:
1. **第一级**把一帧雷达点云(若干带多普勒的目标)压缩成一个 3 维**雷达系自速度** `v_S`;
2. **第二级**把 `v_S` 作为速度观测,紧耦合进 FAST-LIO 的 23 维 ESKF,连带修正姿态与陀螺零偏。

## 1. 第一级:雷达自速度估计

### 1.1 输入数据结构

驱动节点把每帧雷达检测整理成 `RadarScan`,其中每个目标 `RadarTarget` 含:

| 字段 | 含义 |
|------|------|
| `azimuth`  | 方位角 A (rad),雷达系水平面内,正前方为 0 |
| `elevation`| 俯仰角 E (rad),2D 雷达填 0 |
| `range`    | 距离 r (m) |
| `doppler`  | 径向速度 V (m/s),目标相对雷达,远离为正(符号由驱动定义) |
| `snr`/`id` | 信噪比 / 跟踪 ID(可选) |

### 1.2 多普勒观测模型(核心物理)

对第 `i` 个目标,先由方位/俯仰角构造**视线单位向量**(line-of-sight,代码 `los()`):

```
        ⎡ cosE_i · cosA_i ⎤
 d_i =  ⎢ cosE_i · sinA_i ⎥          (2D 雷达 E_i = 0 ⇒ d_i = [cosA_i, sinA_i, 0]ᵀ)
        ⎣ sinE_i          ⎦
```

**关键事实**:静止目标固定于世界系,它相对雷达的速度就是雷达自速度的相反数 `−v_S`。
多普勒只能测到这个相对速度在视线方向上的投影:

```
   V_i = s · (d_iᵀ · v_S)            s = doppler_sign = ±1 (驱动符号约定)
```

令量测 `b_i = s · V_i`,得到对每个静止目标都成立的**线性约束**:

```
   ┌─────────────────────┐
   │   b_i = d_iᵀ · v_S   │     (一个目标 → 一条关于 v_S 的线性方程)
   └─────────────────────┘
```

### 1.3 超定线性方程组

一帧内取距离落在 `[min_range, max_range]` 的 N 个有效目标,堆叠成:

```
   ⎡ d_1ᵀ ⎤            ⎡ b_1 ⎤
   ⎢  ⋮   ⎥ · v_S  =   ⎢  ⋮  ⎥       即   M · v_S = b      (M ∈ ℝ^{N×3},  b ∈ ℝ^N)
   ⎣ d_Nᵀ ⎦            ⎣ b_N ⎦
```

理论上 3 个不共面的视线向量(2D 是 2 个不共线)即可解出 `v_S`。
但真实场景含**运动目标**(行人、车辆),它们不满足上式,是离群点,需鲁棒估计。

### 1.4 RANSAC 剔除动目标

每次迭代(共 `ransac_iter` 次):

1. 随机抽 `min_pts` 个目标(2D 取 2,3D 取 3),组成方阵 `M_k`、向量 `b_k`;
2. 若 `|det(M_k)| < 1e-6`(视线近共线,采样退化)⇒ 跳过;
3. 解候选速度  `v⁽ᵏ⁾ = M_k⁻¹ · b_k`(代码用 colPivHouseholderQr,数值稳定);
4. 合理性检查:`‖v⁽ᵏ⁾‖ > max_speed` ⇒ 丢弃;
5. 统计内点数 —— 残差满足下式者计为内点:

```
   | d_iᵀ · v⁽ᵏ⁾ − b_i |  <  τ          τ = inlier_thresh (默认 0.15 m/s)
```

保留内点最多的候选 `v_S*`。原理:静止目标占多数且彼此一致,
动目标随机散布、互不一致,自然被排除在最大一致集之外。

### 1.5 全内点最小二乘精解

用 `v_S*` 选出全部内点,组成 `M_f, b_f`,求加权(此处等权)最小二乘:

```
   v_S = argmin_v ‖ M_f·v − b_f ‖²  =  (M_fᵀ M_f)⁻¹ M_fᵀ b_f
```

(代码用 QR 分解求解,比正规方程更稳定。)
结果以 `geometry_msgs/TwistStamped` 发布到 `/radar_velocity`,时间戳沿用雷达帧。
**这就是雷达系下的载体对地自速度 `v_S`。**

> 帧内有效目标数不足(`N < max(min_pts, min_inliers)`)或内点数 `< min_inliers` ⇒ 丢弃该帧,不发布。

## 2. 第二级:ESKF 速度紧耦合

### 2.1 状态向量(23 维流形,见 `use-ikfom.hpp`)

```
 x = [  p ,  R , R_LI , t_LI ,  v  ,  b_g ,  b_a ,  g  ]
      0:2  3:5  6:8    9:11  12:14 15:17 18:20  21:22
```

- `p` 位置、`R∈SO(3)` IMU→世界旋转、`v` **世界系**速度;
- `R_LI / t_LI` 激光→IMU 外参、`b_g/b_a` 陀螺/加速度零偏、`g` 重力(S2 流形,2 自由度);
- 下标即误差状态在协方差 `P ∈ ℝ^{23×23}` 中的索引,下面雅可比的 block 位置全部对应它。

雷达相对 IMU 外参:旋转 `R_IS`、平移(杆臂)`t_IS`(`config/mid360.yaml` 的
`radar/extrinsic_R`、`extrinsic_T`,含机头 45° 安装角,**必须按实测标定**)。

### 2.2 观测模型:世界系速度 → 雷达系速度

把当前状态预测成"雷达此刻应测到的自速度":

```
   ┌────────────────────────────────────────────────┐
   │  h(x) = R_ISᵀ · ( Rᵀ·v  +  ω × t_IS )           │
   └────────────────────────────────────────────────┘
```

逐项含义:
- `Rᵀ·v`:世界系速度旋转到 IMU 系;
- `ω × t_IS`:**杆臂引起的旋转线速度**——雷达不在 IMU 原点,载体旋转时雷达处有附加线速度,
  `ω = ω_meas − b_g` 为去偏角速度(取本帧末尾 IMU 角速度);
- 左乘 `R_ISᵀ`:IMU 系再旋转到雷达系。

**残差**:

```
   r = v_S^meas − h(x)         (v_S^meas 即第一级输出的 /radar_velocity)
```

### 2.3 观测雅可比 H = ∂h/∂x  (3 × 23,仅三块非零)

```
   ∂h/∂v   = R_ISᵀ · Rᵀ                  → 列 12:14   (速度块)
   ∂h/∂δθ  = R_ISᵀ · [ Rᵀ·v ]×           → 列 3:5     (姿态块)
   ∂h/∂b_g = R_ISᵀ · [ t_IS ]×           → 列 15:17   (陀螺零偏块)
```

其中 `[·]×` 为反对称(叉乘)矩阵。推导要点:
- 速度块:`h` 对 `v` 线性,直接得 `R_ISᵀ Rᵀ`;
- 姿态块:对旋转扰动 `R ← R·Exp(δθ)` 求导,`∂(Rᵀv)/∂δθ = [Rᵀv]×`;
- 零偏块:`ω` 含 `−b_g`,`∂(ω×t_IS)/∂b_g = ∂(−(t_IS×ω))/∂b_g` 整理得 `[t_IS]×`。

> **符号约定(代码注释明确标注的修正)**:这里必须用 `∂h/∂x`(正号),
> 与更新式 `dx = K·r`、`x ⊞ dx`(boxplus 加法)配套。
> 早期版本误用 `∂(z−h)/∂x` 多了一个负号,会令修正量反向、把滤波器踢发散。

> `radar/vel_only = true` 时只保留速度块,姿态块/零偏块置零 ——
> 雷达只纠正速度,不触碰姿态与零偏(弱耦合,防止高信任时交叉项把姿态带崩)。

### 2.4 标准 EKF 更新

观测噪声(对角):

```
   R_cov = diag( vel_cov·α ,  vel_cov·α ,  vel_cov_z )
```

- `vel_cov` 水平速度噪声(默认 ≈ (0.2 m/s)²),越小越信任雷达;
- `vel_cov_z` 对 2D 雷达设极大(1e6)⇒ 垂直方向几乎不约束;
- `α` 为自适应放大因子(见 2.6)。

新息协方差、增益与流形更新:

```
   S  = H P Hᵀ + R_cov
   K  = P Hᵀ S⁻¹                          (23 × 3)
   dx = K · r
   x  ← x ⊞ dx                            (boxplus,在流形上更新)
   P  ← (I − K H) P
   P  ← ½ (P + Pᵀ)                         (强制对称,防数值退化)
```

### 2.5 卡方门控(挡粗差 / RANSAC 漏检的动目标)

更新前先算新息的马氏距离:

```
   χ² = rᵀ S⁻¹ r
```

`χ² > chi2_thr`(默认 16,对应 3 自由度)⇒ **拒绝本次观测**,直接返回,不更新。

### 2.6 自适应降权:抗自转 / 悬停的杆臂污染

当载体几乎不平移、只原地旋转时,`h` 几乎全由杆臂项 `ω×t_IS` 主导,
而 `t_IS` 的标定误差会被放大成"幽灵速度"污染状态。对策是先剔除杆臂项、估计真实平移:

```
   v_trans = v_S^meas − R_ISᵀ·(ω × t_IS)         v_t = ‖v_trans‖
```

用平滑软门限调节信任度(`v_min = min_trans_vel`,默认 0.3):

```
   w = v_t² / ( v_t² + v_min² )  ∈ (0,1]          α = 1 / w   (噪声放大倍数)
```

- 平移大(`v_t ≫ v_min`)⇒ `w→1`,`α→1`,正常信任雷达;
- 平移趋零 ⇒ `w→0`,`α` 放大到上千倍,雷达观测几乎被忽略。

### 2.7 退化感知解耦更新(`radar/robust = true`,抗强退化的核心)

**问题**:强退化(长隧道)时,FAST-LIO 因 EKF 不一致,把沿轴速度的后验 `P_vv`
算得过小(过度自信)⇒ 标准增益 `K = P Hᵀ S⁻¹ ≈ 0`,雷达"救不动";
若强行调小 `R_cov`,`H` 的姿态/零偏交叉项又会把姿态带发散。

**解法**:用**速度协方差下限** + **只更新速度**,把雷达更新降为一个独立的 3×3 子问题:

```
   H_h = R_ISᵀ · Rᵀ                       (只取速度块)
   P_v = I₃ · pvel_floor                  (速度不确定性下限,不用塌掉的 P)
   S_v = H_h P_v H_hᵀ + R_cov
   χ²_v = rᵀ S_v⁻¹ r        > chi2_thr ⇒ 拒绝
   K_v = P_v H_hᵀ S_v⁻¹                    (3 × 3)
   v   ← v + K_v · r                       (只修正世界系速度)
   P_vv ← (I₃ − K_v H_h) P_v              (只回写 P[12:14, 12:14])
```

- `pvel_floor`(`radar/pvel_floor`)是人为设定的速度不确定度下限,**反映退化方向真实的不确定**,
  使增益不被塌缩的 `P` 压成 0;
- **切断对姿态/零偏的交叉修正**,既保证退化方向雷达能动,又不污染姿态。
- 稳健性与能动性兼得。

### 2.8 时间对齐与触发时机

- 雷达速度异步到达,`radar_vel_cbk()` 缓存进 `radar_time_buffer / radar_vel_buffer`;
- 融合点在**激光 IESKF 更新之后**:每帧激光结束(`lidar_end_time`)时,
  在缓存里找时间戳最近(差 `< time_tol`,默认 0.05 s)的一条速度;
- 取本帧末尾 IMU 角速度做杆臂补偿,调用 `fuse_radar_velocity(v_radar, gyr)`;
- 过期(早于当前帧 1 s)的缓存数据被清理。

## 3. 输出

ESKF 更新后的状态即里程计输出:

- `/Odometry`(`nav_msgs/Odometry`):`pose` 为世界系位姿,`twist.linear` 已填**世界系速度** `v`;
- `/cloud_registered`、`/path` 等照常发布;
- 下游(如 EGO-Planner)直接复用 `/Odometry` 与点云,坐标系对齐 `camera_init`/`body`。

## 4. 全链路数学总览

```
 多普勒        第一级估计              观测模型                         ESKF 更新
 V_i=d_iᵀv_S  ──RANSAC+LSQ──►  v_S  ──►  h = R_ISᵀ(Rᵀv + ω×t_IS)  ──►  x ⊞ K(v_S − h)
                                                                        + 卡方门控
                                                                        + 自适应降权
                                                                        + 退化解耦
```

- **第一级**:把"点云 + 多普勒"通过线性最小二乘压成 3 维速度,RANSAC 保证对动目标鲁棒;
- **第二级**:把该速度作为 ESKF 速度观测,雅可比正确传播到速度/姿态/陀螺零偏,
  辅以**卡方门控**(抗异常)、**自适应降权**(抗自转杆臂)、**退化解耦**(抗强退化)三重保护;
- **净效果**:激光在退化方向(隧道轴向)缺失的约束,由雷达多普勒速度直接补上,抑制沿轴漂移。

| 参数 | 含义 | 默认 |
